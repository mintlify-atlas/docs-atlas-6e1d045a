---
title: Artifact Management Tools
description: Tools for managing files and data within agent sessions
---

Artifact management tools allow agents to create, read, update, and manage files within user sessions. Artifacts persist across conversation turns and can contain any type of data.

## Tool: list_artifacts

Lists all available data artifact filenames and their versions for the current session.

### Usage

```python
from solace_agent_mesh.agent.tools.builtin_artifact_tools import list_artifacts

result = await list_artifacts(tool_context=context)

if result.status == "success":
    artifacts = result.data["artifacts"]
    for artifact in artifacts:
        print(f"File: {artifact['filename']}")
        print(f"Versions: {artifact['versions']}")
        print(f"Metadata: {artifact['metadata_summary']}")
```

### Response Format

```json
{
  "status": "success",
  "message": "Found 3 artifacts.",
  "data": {
    "artifacts": [
      {
        "filename": "sales_data.csv",
        "versions": [0, 1, 2],
        "metadata_summary": {
          "description": "Monthly sales data",
          "type": "text/csv",
          "size": 15840,
          "schema_type": "csv"
        }
      }
    ]
  }
}
```

## Tool: load_artifact

Loads the content or metadata of a specific artifact version.

### Parameters

<ParamField path="filename" type="string" required>
  The name of the artifact to load. May contain embeds like `«artifact_meta:latest_csv_file»`.
</ParamField>

<ParamField path="version" type="integer" required>
  The specific version number to load (0-indexed). Use "latest" for most recent version.
</ParamField>

<ParamField path="load_metadata_only" type="boolean" default={false}>
  If true, loads only the metadata JSON without content.
</ParamField>

<ParamField path="max_content_length" type="integer" default={null}>
  Maximum character length for text content (100-100,000). Uses app configuration if not specified.
</ParamField>

<ParamField path="include_line_numbers" type="boolean" default={false}>
  If true, prefix each line with 1-based line number followed by TAB character.
</ParamField>

### Usage Example

```python
from solace_agent_mesh.agent.tools.builtin_artifact_tools import load_artifact

# Load latest version of artifact
result = await load_artifact(
    filename="report.md",
    version="latest",
    max_content_length=50000,
    include_line_numbers=True,
    tool_context=context
)

if result.status == "success":
    content = result.data["content"]
    version_loaded = result.data["version"]
    mime_type = result.data["mime_type"]
```

### Response Format

```json
{
  "status": "success",
  "message": "Loaded artifact report.md",
  "data": {
    "filename": "report.md",
    "version": 2,
    "mime_type": "text/markdown",
    "content": "1\tMarch Sales Report\n2\t==================\n...",
    "size_bytes": 2048,
    "metadata": {
      "description": "Monthly sales report",
      "created_at": "2024-03-01T10:00:00Z"
    }
  }
}
```

## Tool: extract_content_from_artifact

Uses an internal LLM to extract or transform content from an artifact based on natural language instructions.

### Parameters

<ParamField path="filename" type="string" required>
  Name of the source artifact to extract content from.
</ParamField>

<ParamField path="extraction_goal" type="string" required>
  Natural language instruction describing what to extract or how to transform the content.
  
  Examples:
  - "Extract all email addresses"
  - "Summarize the key findings"
  - "Convert to JSON format"
  - "List all products with price > $100"
</ParamField>

<ParamField path="version" type="string" default="latest">
  Version of the source artifact. Can be integer or "latest".
</ParamField>

<ParamField path="output_filename_base" type="string" default={null}>
  Optional base name for saving extracted content as a new artifact.
</ParamField>

### Usage Example

```python
from solace_agent_mesh.agent.tools.builtin_artifact_tools import extract_content_from_artifact

# Extract data from CSV artifact
result = await extract_content_from_artifact(
    filename="sales_data.csv",
    extraction_goal="Extract top 10 products by revenue and format as a table",
    version="latest",
    output_filename_base="top_products",
    tool_context=context
)

if result.status == "success":
    extracted_content = result.data.get("extracted_content")
    # Or if saved as artifact:
    output_filename = result.data.get("output_filename")
    output_version = result.data.get("output_version")
```

### Configuration

The extraction tool can use a different LLM model than the main agent:

```yaml
builtin_tools:
  - name: extract_content_from_artifact
    config:
      model:
        model: gpt-4o-mini  # Use faster/cheaper model for extraction
        temperature: 0.1
      supported_binary_mime_types:
        - "application/pdf"
        - "image/*"
      tool_output_save_threshold_bytes: 2048
      tool_output_llm_return_max_bytes: 4096
```

<ParamField path="supported_binary_mime_types" type="array" default={[]}>
  List of binary MIME type patterns that can be processed (e.g., `["application/pdf", "image/*"]`).
</ParamField>

<ParamField path="model" type="object" default={null}>
  Optional LLM model configuration for extraction. Can be a model string or full config dict.
</ParamField>

### Response Format

```json
{
  "status": "success",
  "message": "Successfully extracted content from 'sales_data.csv'",
  "data": {
    "extracted_content": "| Product | Revenue |\n|---------|---------|\n| Widget A | $45,230 |",
    "output_filename": "top_products_extract.txt",
    "output_version": 0,
    "saved_as_artifact": true
  }
}
```

## Tool: apply_embed_and_create_artifact

Resolves an embed directive (including modifiers and formatting) and saves the result as a new artifact.

### Parameters

<ParamField path="output_filename" type="string" required>
  The desired name for the new artifact.
</ParamField>

<ParamField path="embed_directive" type="string" required>
  The full embed directive string, e.g., `«artifact_content:data.csv>>>format:json»`
  
  Supports:
  - Modifiers: `limit:100`, `offset:50`, `sample:10`
  - Formats: `html`, `json`, `csv`, `datauri`
  - Chaining: `«artifact_content:data.json>>>limit:100>>>format:csv»`
</ParamField>

<ParamField path="output_metadata" type="object" default={null}>
  Optional metadata dictionary for the new artifact.
</ParamField>

### Usage Example

```python
from solace_agent_mesh.agent.tools.builtin_artifact_tools import apply_embed_and_create_artifact

# Transform CSV to JSON using embed
result = await apply_embed_and_create_artifact(
    output_filename="sales_data.json",
    embed_directive="«artifact_content:sales_data.csv>>>format:json»",
    output_metadata={
        "source": "sales_data.csv",
        "transform": "CSV to JSON"
    },
    tool_context=context
)

if result.status == "success":
    print(f"Created: {result.data['output_filename']} v{result.data['output_version']}")
```

## Artifact Metadata

All artifacts include metadata that provides context and schema information:

```json
{
  "description": "Monthly sales data for Q1 2024",
  "source": "uploaded_by_user",
  "mime_type": "text/csv",
  "size_bytes": 15840,
  "created_at": "2024-03-01T10:00:00Z",
  "schema": {
    "type": "csv",
    "inferred": true,
    "columns": [
      {"name": "product_id", "type": "integer"},
      {"name": "product_name", "type": "string"},
      {"name": "revenue", "type": "number"}
    ]
  }
}
```

## Best Practices

<AccordionGroup>
  <Accordion title="Use descriptive filenames">
    Choose clear, descriptive names for artifacts that indicate their content and purpose.
    
    ```python
    # Good
    filename = "q1_2024_sales_summary.csv"
    
    # Avoid
    filename = "data.csv"
    ```
  </Accordion>

  <Accordion title="Include metadata">
    Always provide descriptive metadata to help track artifact provenance and purpose.
    
    ```python
    metadata = {
        "description": "Sales data aggregated by product category",
        "source": "database_query",
        "query_date": "2024-03-01",
        "aggregation_level": "category"
    }
    ```
  </Accordion>

  <Accordion title="Version management">
    Use version numbers to track artifact evolution. Versions are 0-indexed and immutable.
    
    ```python
    # Load specific version for reproducibility
    result = await load_artifact(filename="report.md", version=2)
    
    # Or always use latest
    result = await load_artifact(filename="report.md", version="latest")
    ```
  </Accordion>
</AccordionGroup>

## Tool: append_to_artifact

Appends a chunk of content to an existing artifact, creating a new version. Useful for building up files incrementally without loading the entire content into memory.

### Parameters

<ParamField path="filename" type="string" required>
  The name of the artifact to append to. May contain embeds.
</ParamField>

<ParamField path="content_chunk" type="string" required>
  The chunk of content to append (max ~3KB recommended). If `mime_type` suggests binary content, this should be base64 encoded. May contain embeds.
</ParamField>

<ParamField path="mime_type" type="string" required>
  The MIME type of the content chunk. Helps determine if base64 decoding is needed. The overall artifact's MIME type is preserved from its latest version.
</ParamField>

### Usage

```python
# Append text to a log file
result = await append_to_artifact(
    filename="application.log",
    content_chunk="[2024-01-15 10:30:45] Server started successfully\n",
    mime_type="text/plain",
    tool_context=context
)

# Append CSV rows incrementally
result = await append_to_artifact(
    filename="transactions.csv",
    content_chunk="TX-12345,2024-01-15,149.99,completed\n",
    mime_type="text/csv",
    tool_context=context
)
```

<Note>
  Keep chunk sizes small (recommended max 3KB) to avoid LLM context window issues. For larger files, make multiple append calls.
</Note>

## Tool: delete_artifact

Deletes all versions of an artifact. Requires explicit confirmation for safety.

### Parameters

<ParamField path="filename" type="string" required>
  The name of the artifact to delete.
</ParamField>

<ParamField path="version" type="integer">
  Reserved for future use. Version-specific deletion is not currently supported and will return an error if specified.
</ParamField>

<ParamField path="confirm_delete" type="boolean">
  Must be set to `true` to actually perform the deletion. If `false` or omitted, returns a confirmation prompt with details about what will be deleted (including version count).
</ParamField>

### Usage

```python
# First call without confirmation shows what will be deleted
result = await delete_artifact(
    filename="temp_data.csv",
    tool_context=context
)
# Returns: "Are you sure you want to delete 'temp_data.csv' (3 versions)? Set confirm_delete=True to proceed."

# Second call with confirmation performs the deletion
result = await delete_artifact(
    filename="temp_data.csv",
    confirm_delete=True,
    tool_context=context
)
# Returns: "Artifact 'temp_data.csv' (3 versions) deleted successfully."
```

<Warning>
  This operation deletes **all versions** of the artifact and cannot be undone. The two-step confirmation process helps prevent accidental deletions.
</Warning>

## Tool: artifact_search_and_replace_regex

Performs search and replace operations on an artifact's text content using either literal string matching or regular expressions. Supports both single replacements and atomic batch replacements.

### Parameters

#### Single Replacement Mode

<ParamField path="filename" type="string" required>
  The name (and optional `:version`) of the artifact to search/replace in.
</ParamField>

<ParamField path="search_expression" type="string">
  The pattern to search for. If `is_regexp` is true, this is treated as a regular expression. Otherwise, it's a literal string. Do not use with `replacements`.
</ParamField>

<ParamField path="replace_expression" type="string">
  The replacement text. For regex mode, supports capture group references using `$1`, `$2`, etc. Use `$$` to insert a literal dollar sign. Do not use with `replacements`.
</ParamField>

<ParamField path="is_regexp" type="boolean" default="false">
  If true, treat `search_expression` as a regular expression. If false, treat as literal string.
</ParamField>

<ParamField path="regexp_flags" type="string">
  Regular expression flags (e.g., `"i"` for case-insensitive, `"m"` for multiline). Only used in regex mode.
</ParamField>

#### Batch Replacement Mode

<ParamField path="replacements" type="array">
  Array of replacement operations to perform atomically. Each object should have `search`, `replace`, `is_regexp`, and optional `regexp_flags` fields.
</ParamField>

### Usage

#### Literal string replacement

```python
# Replace all occurrences of a literal string
result = await artifact_search_and_replace_regex(
    filename="config.yaml",
    search_expression="localhost:8080",
    replace_expression="production-server.com:443",
    is_regexp=False,
    tool_context=context
)
```

#### Regular expression replacement

```python
# Replace dates in MM/DD/YYYY format with YYYY-MM-DD
result = await artifact_search_and_replace_regex(
    filename="report.txt",
    search_expression=r"(\d{2})/(\d{2})/(\d{4})",
    replace_expression="$3-$1-$2",
    is_regexp=True,
    tool_context=context
)
```

#### Batch replacements

```python
# Perform multiple replacements atomically
result = await artifact_search_and_replace_regex(
    filename="source_code.py",
    replacements=[
        {
            "search": "old_function_name",
            "replace": "new_function_name",
            "is_regexp": False
        },
        {
            "search": r"# TODO:.*$",
            "replace": "# DONE:",
            "is_regexp": True,
            "regexp_flags": "m"
        }
    ],
    tool_context=context
)
```

<Info>
  Batch mode is more efficient than multiple single replacement calls because it processes all replacements in a single atomic operation, creating only one new version.
</Info>

## Error Handling

Common error scenarios and how to handle them:

```python
result = await load_artifact(filename="data.csv", version=0, tool_context=context)

if result.status == "error":
    if "not found" in result.message.lower():
        # Artifact doesn't exist
        print("Artifact not found")
    elif "invalid" in result.message.lower():
        # Invalid parameters
        print(f"Invalid parameters: {result.message}")
    else:
        # Other error
        print(f"Error: {result.message}")
```
